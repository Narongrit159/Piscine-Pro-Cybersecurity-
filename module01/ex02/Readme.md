# Exercise 02 - Server-Side Request Forgery (SSRF) + Local File Inclusion (LFI)

## ประเภทของช่องโหว่

- Server-Side Request Forgery (SSRF)
- Local File Inclusion (LFI)

## คำอธิบาย

SSRF เกิดจากการที่ server รับ URL จากผู้ใช้แล้วไป fetch ให้โดยตรง
โดยไม่มีการตรวจสอบปลายทางอย่างเหมาะสม

LFI คือช่องโหว่ที่ทำให้ attacker สามารถระบุ path ของไฟล์ในระบบ
เพื่อให้แอปพลิเคชันอ่านไฟล์เหล่านั้นมาแสดงผล

ในแลปนี้ แอปพลิเคชันรับ URL จากผู้ใช้และนำไป fetch โดยตรง
ทำให้สามารถใช้ protocol `file://` หรือ path ภายในเครื่อง
เพื่ออ่านไฟล์ระบบได้

## หลักการทำงานโดยย่อ

1. ผู้ใช้กรอก URL
2. Server ทำ request ไปตาม URL ที่ได้รับ
3. เนื้อหาที่ fetch ได้ถูกนำมาแสดงบนหน้าเว็บ
4. attacker ใช้ URL ที่ชี้ไปยังไฟล์ภายในเครื่อง

## ผลกระทบ

- อ่านไฟล์ภายในระบบ เช่น /etc/passwd
- เปิดเผยข้อมูลลับ
- อาจนำไปสู่การโจมตีบริการภายในเครือข่าย
