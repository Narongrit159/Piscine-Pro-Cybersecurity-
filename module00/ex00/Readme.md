# Exercise 00 - Cross-Site Scripting (XSS)

## ประเภทของช่องโหว่

Cross-Site Scripting (XSS)

## คำอธิบาย

ช่องโหว่ XSS เกิดจากการที่เว็บแอปพลิเคชันนำข้อมูลจากผู้ใช้งานมาแสดงผลโดยตรง
โดยไม่มีการตรวจสอบหรือ escape ข้อมูล ทำให้ attacker สามารถฝังโค้ด JavaScript
ลงไปในหน้าเว็บได้

เมื่อผู้ใช้เปิดหน้าเว็บดังกล่าว เบราว์เซอร์จะรัน JavaScript ที่ถูกฝังไว้ทันที

ในแลปนี้ หน้าเว็บนำค่าจาก input มาใส่ลงใน innerHTML โดยตรง
รวมถึงมีการสร้าง script จาก input ของผู้ใช้ ทำให้สามารถรันโค้ดใดๆ ได้

## ผลกระทบ

- สามารถขโมย cookie หรือ session ของผู้ใช้
- สามารถปลอมหน้าเว็บเพื่อหลอกเอาข้อมูล
- สามารถรันโค้ดในบริบทของเหยื่อได้

## วิธีตรวจพบ

ทำการใส่ HTML หรือ JavaScript ลงในช่อง input  
หากหน้าเว็บนำไป render หรือ execute ได้ แสดงว่ามีความเสี่ยงต่อ XSS
