# การแปลงข้อมูลกลับโดยไม่ปลอดภัย (Insecure Deserialization)

## ภาพรวม

โปรเจคนี้เป็นการสาธิตช่องโหว่ด้านความปลอดภัยจากการแปลงข้อมูลกลับเป็นอ็อบเจ็กต์
โดยไม่ตรวจสอบความปลอดภัย

เว็บแอปพลิเคชันรับข้อมูลจากผู้ใช้ในรูปแบบ serialized object ก็คือ
ข้อมูลที่แปลงมาเป็น object เพื่อส่งต่อ
และนำไปแปลงกลับด้วย `pickle.loads()` โดยตรง
ทำให้ผู้โจมตีสามารถฝังคำสั่งระบบไว้ในข้อมูลที่ส่งเข้าไปได้

เมื่อฝั่งเซิร์ฟเวอร์ทำการแปลงข้อมูลกลับ
คำสั่งที่ถูกฝังไว้จะถูกรันทันที

ส่งผลให้เกิดการสั่งรันโค้ดจากระยะไกล (Remote Code Execution)

---

## ประเภทของช่องโหว่

- การแปลงข้อมูลกลับโดยไม่ปลอดภัย (Insecure Deserialization)
- การสั่งรันโค้ดจากระยะไกล (Remote Code Execution)

---

## หลักการโจมตี

1. สร้างอ็อบเจ็กต์ Python ที่กำหนดเมธอด `__reduce__`
2. ภายในเมธอดระบุให้เรียกคำสั่งระบบ
3. แปลงอ็อบเจ็กต์เป็นข้อมูลด้วย pickle
4. เข้ารหัสเป็น base64
5. ส่งข้อมูลนี้ไปยังเว็บ
6. เว็บนำข้อมูลไปแปลงกลับ
7. คำสั่งระบบถูกเรียกใช้งานทันที
